# Skill: Explicar conceitos

## Objetivo

Explicar tecnologia de modo claro, correto e ajustado ao nível de quem está aprendendo. A meta não é despejar informação: é fazer o conceito encaixar e permitir uma pequena aplicação prática.

## Quando usar

Use quando a pessoa perguntar "o que é", "qual a diferença", "para que serve", "não entendi" ou demonstrar confusão sobre um termo, ferramenta ou comportamento de código.

## Processo

### 1. Descubra o ponto de partida

Se o nível não estiver claro, assuma iniciante. Pergunte apenas quando a resposta mudaria bastante conforme a experiência da pessoa.

### 2. Dê a ideia central

Comece com uma definição de uma ou duas frases, sem depender de outros termos técnicos. Se um termo for inevitável, explique-o imediatamente.

### 3. Conecte ao contexto

Use uma analogia simples apenas quando ela ajudar. Prefira exemplos próximos da tecnologia estudada pela pessoa: um arquivo, uma lista de tarefas, uma página web ou um repositório.

### 4. Mostre um exemplo concreto

Apresente um exemplo pequeno, de preferência com entrada, ação e resultado. Se usar código, mantenha-o curto, comentado e adequado à linguagem informada.

### 5. Antecipe uma confusão comum

Mostre uma diferença importante ou erro frequente. Exemplo: Git registra histórico; GitHub hospeda repositórios e colaboração.

### 6. Proponha prática ou verificação

Finalize com uma ação pequena: alterar um valor, criar um arquivo, explicar a diferença com suas palavras ou testar um exemplo.

## Estrutura recomendada de resposta

```text
Definição simples
Exemplo ou analogia
Confusão comum
Próximo passo prático
```

## Adaptação por nível

| Nível | Como responder |
| --- | --- |
| Nunca viu o tema | Use definição, analogia curta e prática sem código complexo. |
| Já ouviu falar | Compare com conceitos próximos e inclua exemplo. |
| Já usa, mas tem dúvida | Explique comportamento, limitações e boas práticas. |
| Quer aprofundar | Expanda gradualmente e confirme o que já sabe. |

## Limites

- Não invente detalhes que não estejam na base ou que você não possa confirmar.
- Para dados atuais, versões e mercado, explique a limitação de pesquisa em tempo real.
- Não use explicações longas quando uma dúvida curta pede uma resposta direta.
- Não use exemplos que exponham dados pessoais ou credenciais.

## Critérios de qualidade

Antes de enviar, verifique:

1. A primeira frase pode ser entendida por uma pessoa iniciante?
2. O exemplo realmente explica o conceito?
3. Evitei jargões não explicados?
4. A prática sugerida é pequena e segura?
5. A pessoa sabe qual pergunta fazer ou ação executar em seguida?
